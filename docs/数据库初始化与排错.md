# 教师工作量系统：数据库初始化与“页面全是 0 / No Data”排错

## 1. 先修复前端请求被 Vite 吃掉的问题（你截图里的 JSON 报错根因）

如果前端在 `localhost:5173` 运行，但没有配置 `/api` 代理，页面请求 `/api/...` 会被前端开发服务器返回 HTML，导致：

- `Unexpected token '<', "<head>" ... is not valid JSON`

本项目已在 `frontend/vite.config.js` 增加代理到 `http://localhost:8080`。重启前端后生效。

## 2. 启动顺序（建议）

1. 先启动 MySQL，确认库名：`teacher_workload_management`
2. 启动后端（8080）
3. 再启动前端（5173）

## 3. 快速自检 SQL（确认“表已建、数据不空”）

```sql
USE teacher_workload_management;

SHOW TABLES;

SELECT COUNT(*) AS admin_cnt FROM admin;
SELECT COUNT(*) AS teacher_cnt FROM teacher;
SELECT COUNT(*) AS type_cnt FROM workload_type;
SELECT COUNT(*) AS workload_cnt FROM workload;
SELECT COUNT(*) AS rule_cnt FROM warning_rule;
SELECT COUNT(*) AS record_cnt FROM warning_record;
```

> 如果 `SHOW TABLES` 里没有业务表，通常是后端没连上这套库，或后端没成功启动。

## 4. 最小可用初始化数据（可直接执行）

```sql
USE teacher_workload_management;

INSERT INTO admin (username, password, name)
SELECT 'admin', '123456', '系统管理员'
WHERE NOT EXISTS (SELECT 1 FROM admin WHERE username = 'admin');

INSERT INTO teacher (teacher_no, password, name, gender, phone, email, department, title, status)
SELECT 'T001', '123456', '张三', '男', '13800138000', 'zhangsan@example.com', '计算机学院', '讲师', 1
WHERE NOT EXISTS (SELECT 1 FROM teacher WHERE teacher_no = 'T001');

INSERT INTO workload_type (type_name, unit_value, remark)
SELECT '教学', 2.00, '教学工作量'
WHERE NOT EXISTS (SELECT 1 FROM workload_type WHERE type_name = '教学');

INSERT INTO workload_type (type_name, unit_value, remark)
SELECT '科研', 1.00, '科研工作量'
WHERE NOT EXISTS (SELECT 1 FROM workload_type WHERE type_name = '科研');
```

## 5. 为什么“我的工作量列表”提示拿不到 teacherId

你当前界面显示“管理员”，管理员账号本身不是教师账号；“我的工作量列表”页需要教师 ID。

- 管理员更适合看“工作量审核 / 统计分析”
- 教师账号（如 `T001`）登录后再看“我的工作量列表”更合理

## 6. 如果你还缺表

本项目后端是 JPA 自动建表模式（`ddl-auto: update`）。若缺表请检查：

- `application.yml` 中 MySQL 地址、用户名、密码是否和你当前库一致
- 后端启动日志是否报连接失败
- 连接的是不是同一个 MySQL 实例（很多同学本地有多个 MySQL）
