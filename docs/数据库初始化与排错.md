# 教师工作量系统：数据库初始化与“页面全是 0 / No Data”排错

## 1. 先修复前端请求被 Vite 吃掉的问题（你截图里的 JSON 报错根因）

如果前端在 `localhost:5173` 运行，但没有配置 `/api` 代理，页面请求 `/api/...` 会被前端开发服务器返回 HTML，导致：

- `Unexpected token '<', "<head>" ... is not valid JSON`

本项目已在 `frontend/vite.config.js` 增加代理到 `http://localhost:8080`。重启前端后生效。

## 2. 启动顺序（建议）

1. 先启动 MySQL，确认库名：`teacher_workload_management`
2. 启动后端（8080）
3. 再启动前端（5173）

## 3. 快速自检 SQL（确认“表已建、数据不空”）

```sql
USE teacher_workload_management;

SHOW TABLES;

SELECT COUNT(*) AS admin_cnt FROM admin;
SELECT COUNT(*) AS teacher_cnt FROM teacher;
SELECT COUNT(*) AS type_cnt FROM workload_type;
SELECT COUNT(*) AS workload_cnt FROM workload;
SELECT COUNT(*) AS rule_cnt FROM warning_rule;
SELECT COUNT(*) AS record_cnt FROM warning_record;
```

> 如果 `SHOW TABLES` 里没有业务表，通常是后端没连上这套库，或后端没成功启动。

## 5. 为什么“我的工作量列表”提示拿不到 teacherId

你当前界面显示“管理员”，管理员账号本身不是教师账号；“我的工作量列表”页需要教师 ID。

- 管理员更适合看“工作量审核 / 统计分析”
- 教师账号（如 `T001`）登录后再看“我的工作量列表”更合理

## 6. 如果你还缺表

本项目后端是 JPA 自动建表模式（`ddl-auto: update`）。若缺表请检查：

- `application.yml` 中 MySQL 地址、用户名、密码是否和你当前库一致
- 后端启动日志是否报连接失败
- 连接的是不是同一个 MySQL 实例（很多同学本地有多个 MySQL）

## 7. 如果你想在 Navicat 手工建表（仅在 JPA 自动建表失败时使用）

> 正常情况下不用手工建表。只有在后端无法自动建表时，才建议执行以下 DDL。

```sql
CREATE TABLE IF NOT EXISTS admin (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL UNIQUE,
  password VARCHAR(100) NOT NULL,
  name VARCHAR(50) NOT NULL,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS teacher (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  teacher_no VARCHAR(50) NOT NULL UNIQUE,
  password VARCHAR(100) NOT NULL,
  name VARCHAR(50) NOT NULL,
  gender VARCHAR(10),
  phone VARCHAR(30),
  email VARCHAR(100),
  department VARCHAR(100),
  title VARCHAR(50),
  status INT DEFAULT 1,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS workload_type (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  type_name VARCHAR(50) NOT NULL UNIQUE,
  unit_value DECIMAL(10,2) NOT NULL,
  remark VARCHAR(255)
);

CREATE TABLE IF NOT EXISTS workload (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  teacher_id BIGINT NOT NULL,
  type_id BIGINT NOT NULL,
  workload_title VARCHAR(100) NOT NULL,
  description VARCHAR(300),
  amount DECIMAL(10,2),
  submit_date DATE,
  status VARCHAR(20) DEFAULT 'PENDING',
  reject_reason VARCHAR(255),
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT fk_workload_teacher FOREIGN KEY (teacher_id) REFERENCES teacher(id),
  CONSTRAINT fk_workload_type FOREIGN KEY (type_id) REFERENCES workload_type(id)
);

CREATE TABLE IF NOT EXISTS warning_rule (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  rule_name VARCHAR(100) NOT NULL,
  min_value DECIMAL(10,2),
  max_value DECIMAL(10,2),
  type_id BIGINT,
  enabled INT DEFAULT 1,
  remark VARCHAR(255)
);

CREATE TABLE IF NOT EXISTS warning_record (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  teacher_id BIGINT NOT NULL,
  rule_id BIGINT NOT NULL,
  warning_message VARCHAR(255) NOT NULL,
  status VARCHAR(20),
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_record_teacher FOREIGN KEY (teacher_id) REFERENCES teacher(id),
  CONSTRAINT fk_record_rule FOREIGN KEY (rule_id) REFERENCES warning_rule(id)
);
```
